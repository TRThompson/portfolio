KEY POINTS:
  • Big-O notation is used to show that the time algorithms take to execute increases as the size of the data set they operate increases

  • To get the Big-O value:
  • - remove all terms except the one with the largest exponent
  • - remove any constant factors

  • Constant complexities take the same number of steps regardless of the size of the data set
  • The steps taken by an algorithm of linear complexity increase at the same rate as the size of the data set
  • Polynomial complexities are those that can be written as O(n^k) where k is a constant integer (constant and linear and both polynomial)
    • Exponential complexities can be written as O(k^n)
  • Logarithmic complexities can be written as O(log n)
  • linear search checks one item at a time until the desired item is found
  • Binary search splits a list in 2 repeatedly and discards half until the searched-for item is found or the list is exhausted
  • Binary search will usually find an item in much fewer checks than in linear search but it requires the list to be sorted
