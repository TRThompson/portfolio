Elements of software development Key points/summary:
  • Software projects can be expensive and without careful managements can go overtime/ budget without being completed
  • Before starting a project a feasibility study should be carried out to see if the project is likely to be successful
  • The requirements specification needs to be an agreement between the company and the end user so everyone involved knows what the project should achieve
  • Testing is carried out to find and eliminate bugs.
  • Destructive testing tries to ‘break’ the program
  • Alpha testing uses people within the company
  • Beta testing requires people outside of the company to test the program
  • Documentation is generated while the system is developed
  • Technical documentation explains how the system works
  • User documentation explains how to use the system
  • Black box testing sees the software as a ‘back box’ with just inputs and outputs
  • White box testing analyses the structure and logic of the program
