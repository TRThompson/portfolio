Chapter 4

KEY TERMS:
  • BRA: branch always. A Jump instruction that's always executed
  • BRP: Branch if the value in the accumulator is positive
  • Sequence: execution of statements/functions one after another - usually forms the bulk of the code in any program
  • Selection: the flow of the program is interrupted and control is passed onto another part of the program, the decision is based upon a boolean
  • Iteration: a section of code is repeated, based upon a boolean
  • Immutable: Unchangeable. An entity that cannot be changed by the program
  • Source code: The code that is written in the programming language. Software is open source if the source code is available
  • Build: Refers to all the actions that a programmer takes to produce a finished working program
KEY POINTS:
  • All programs can be based on sequences, branches (selection) and iteration
  • Most programming languages have their own ways of enabling these constructs making them easier for the programmer
  • Selection is sometimes referred to as ‘branching’ and iteration means repetition of some of the process/code/program.
  • Variables can be altered during the running of a program
  • Variables have a data type that is usually fixed for the duration of the program
  • Variables are declared in advance in some languages but not in others
  • Variables can be global or local
  • Safer to use local variables where possible to avoid conflicts
